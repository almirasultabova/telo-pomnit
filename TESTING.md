## Часть 1: Лендинги

### Как открыть

Никаких зависимостей нет. Просто откройте файл в браузере:

```
landing_final.html  ← основной, тестировать в первую очередь
```

Или через локальный сервер (рекомендуется — корректнее работают шрифты и ссылки):

```bash
# Python (есть на любом Mac/Linux, обычно есть на Windows)
python -m http.server 8080
# затем открыть http://localhost:8080/landing_final.html
```

### Что проверить

#### Визуальное

- [ ] Страница открывается без ошибок в консоли (F12 → Console)
- [ ] Анимированные элементы работают (блобы, breathing-кружок, пульсирующие кольца)
- [ ] Все секции видны, нет обрезанного текста

#### Адаптивность

Проверить в DevTools (F12 → иконка телефона) при следующих размерах:

- [ ] Навигация: гамбургер-меню появляется на ≤ 768px и работает (открывает/закрывает)

## Часть 2: Telegram Mini App (`tg-app/`)

### Как открыть (без Telegram)

Приложение работает в браузере напрямую — SDK Telegram не ломает приложение при его отсутствии.

```
Открыть файл: tg-app/index.html
```

Или через локальный сервер:
```bash
python -m http.server 8080
# http://localhost:8080/tg-app/
```

### Порядок загрузки скриптов (из index.html)

```
1. telegram-web-app.js (CDN Telegram)
2. js/data.js
3. js/storage.js
4. js/app.js
```

Если скрипты перепутаны — приложение не запустится.

### Сброс данных перед тестом

Данные хранятся в `localStorage`. Для чистого теста:

```
F12 → Application → Local Storage → localhost → выделить всё → Delete
```

Ключи которые нужно сбросить:
- `tp_diary_entries`
- `tp_diag_result`
- `tp_onboarding_done`
- `tp_offer_seen`
- `tp_attended`

### Сценарии тестирования

#### Сценарий 1: Первый запуск (онбординг)

**Предусловие:** localStorage пуст (сброшен выше)

1. Открыть `tg-app/index.html`
2. Должен появиться splash-экран (🫀 «Тело помнит») — показывается 1.6 секунды
3. После splash — экран онбординга с кнопкой «Начать»
4. Нажать «Начать» → перейти на главный экран (вкладка «Дневник»)
5. После первого запуска должна появиться модалка-оффер («Добро пожаловать в поток»)

- [ ] Splash появляется и уходит через ~1.6 сек
- [ ] Онбординг показывается при первом запуске
- [ ] При повторном открытии (без сброса) онбординг НЕ показывается — сразу главный экран
- [ ] Модалка-оффер показывается один раз, после закрытия больше не появляется

---

#### Сценарий 2: Запись в дневник

1. На главном экране (вкладка «Дневник») нажать кнопку добавления записи
2. Экран «Карта тела» — выбрать зону (например «Грудь»)
   - Переключить вид «Спереди» / «Сзади» — зоны меняются
   - Кнопка «Продолжить» активируется только после выбора зоны
3. Экран «Ощущения» — выбрать одно или несколько из 8 вариантов:
   - Зажатость, Тяжесть, Пульсация, Онемение, Жжение, Дрожь, Пустота, Тепло
   - Кнопка «Продолжить» активируется только после выбора хотя бы одного
4. Экран «Заметка» — ввести текст или нажать «Пропустить»
5. Экран «Запись сохранена» — появляется ✓ и счётчик стрика 🔥
6. Нажать «На главную» — вернуться на дневник

- [ ] Зоны тела кликабельны, при выборе подсвечиваются
- [ ] Переключение Спереди/Сзади работает корректно
- [ ] Мультиселект ощущений работает (можно выбрать несколько)
- [ ] «Пропустить» на экране заметки сохраняет запись без текста
- [ ] После сохранения запись появляется в разделе «Последние записи» на главном экране
- [ ] Стрик (счётчик дней подряд) увеличивается

## Что делать при ошибке

1. **Белый экран** — открыть F12 → Console, найти красную ошибку
2. **Скрипт не загрузился** — проверить порядок тегов `<script>` в `tg-app/index.html` (data.js → storage.js → app.js)
3. **Данные не сохраняются** — проверить F12 → Application → Local Storage, убедиться что ключи `tp_*` появляются
4. **Сломался стрик** — стрик считается через `Storage.getStreak()` в `tg-app/js/storage.js:52`
5. **Зоны тела не кликаются** — проверить что зона не помечена `selectable: false` в `tg-app/js/data.js`
