## Часть 1: Лендинг

### Как открыть локально

```bash
python -m http.server 8080
# http://localhost:8080/landing_final.html
```

### Что проверить

#### Визуальное
- [ ] Страница открывается без ошибок (F12 → Console)
- [ ] Анимированные элементы работают
- [ ] Все секции видны, нет обрезанного текста

#### Адаптивность (F12 → иконка телефона)

| Размер | Что должно быть |
|---|---|
| **375px** (iPhone SE) | Все grid-сетки в 1 колонку, кнопки на всю ширину |
| **768px** (iPad) | Гамбургер-меню в навигации |
| **1280px** (десктоп) | Полная сетка, горизонтальная навигация |

#### Оплата
- [ ] Кнопки «Записаться» и «Участвовать» открывают модальное окно с полем email
- [ ] После ввода email и нажатия «Перейти к оплате» — редирект на страницу ЮКассы
- [ ] После оплаты — редирект на `thanks.html`
- [ ] На email участницы приходит письмо со ссылкой на бота
- [ ] До 22 марта цена 12 000 ₽, после — 15 000 ₽
- [ ] После 27 марта кнопки меняются на «Мест нет»

## Часть 2: Telegram Mini App

### Как открыть (без Telegram)

```
Открыть: tg-app/index.html
```

Или через локальный сервер:
```bash
python -m http.server 8080
# http://localhost:8080/tg-app/
```

### Порядок загрузки скриптов (`index.html`)

```
1. telegram-web-app.js (CDN Telegram)
2. js/data.js
3. js/storage.js
4. js/app.js
```

Если скрипты перепутаны — приложение не запустится.

### Сброс данных перед тестом

```
F12 → Application → Local Storage → localhost → выделить всё → Delete
```

Ключи для сброса:
- `tp_diary_entries`
- `tp_diag_result`
- `tp_onboarding_done`
- `tp_offer_seen`
- `tp_attended`

### Сценарии тестирования

#### Сценарий 1: Первый запуск

**Предусловие:** localStorage пуст

1. Открыть `tg-app/index.html`
2. Splash-экран (~1.6 сек) → онбординг → главный экран
3. После первого запуска — модалка-оффер «Добро пожаловать в поток»

- [ ] Splash появляется и уходит
- [ ] Онбординг при первом запуске, при повторном — сразу главный экран
- [ ] Модалка-оффер показывается один раз
- [ ] Кнопка «Вступить в поток» ведёт на chatUrl из `data.js`

---

#### Сценарий 2: Запись в дневник

1. Главный экран → кнопка добавления записи
2. **Карта тела** — выбрать зону (спереди или сзади)
3. **Ощущения** — выбрать одно или несколько
4. **Заметка** — ввести текст или «Пропустить»
5. Экран «Запись сохранена» → «На главную»

- [ ] Зоны на PNG-изображении тела кликабельны и подсвечиваются
- [ ] Переключение «Спереди» / «Сзади» меняет вид и зоны
- [ ] Ладони и стопы — кликабельные зоны
- [ ] Запись появляется в «Последние записи»
- [ ] Стрик увеличивается

---

#### Сценарий 3: Стоп-реакция

1. Вкладка «Стоп-реакция»
2. Заполнить: ситуация, тип реакции (freeze/fight/flight/fawn), зона, интенсивность
3. Сохранить

- [ ] Форма заполняется и сохраняется
- [ ] Запись появляется в истории

---

#### Сценарий 4: Чекин

1. Вкладка «Чекин» (или «Новая реакция»)
2. Оценить тело по шкале, выбрать зону напряжения, настроение

- [ ] Чекин сохраняется
- [ ] Сегодняшний чекин отображается корректно

---

#### Сценарий 5: Диагностика паттернов

1. Вкладка «Мой путь»
2. Пройти 10 вопросов

- [ ] Прогресс-бар обновляется
- [ ] Результат: один из 4 паттернов (freeze/fawn/fight/flight)
- [ ] Есть кнопка «Пройти заново»

---

#### Сценарий 6: AI-чат

**Без активного зачисления:**
- [ ] Показывается locked-экран с текстом «AI-ассистент доступен участницам программы»
- [ ] Поле ввода скрыто

**С активным зачислением (статус `active`):**
- [ ] Приветственное сообщение
- [ ] Можно отправить сообщение и получить ответ

#### Сценарий 7: Вкладка «Мой поток»

1. Нижняя навигация → «Мой поток»

- [ ] Карточка пользователя (имя из Telegram или заглушка)
- [ ] Статистика (записей, стрик)
- [ ] Кнопка «Войти в групповой чат» ведёт на chatUrl
- [ ] Расписание 9 встреч с 19 марта по 16 апреля 2026
- [ ] При нажатии на встречу — нижний лист с деталями
- [ ] Карточки ведущих (Анастасия, Альмира) с ссылками в Telegram

---

#### Сценарий 8: Навигация

- [ ] Нижняя навигация (3 вкладки) переключается
- [ ] Кнопка «Назад» в браузере возвращает на предыдущий экран
- [ ] Нет экранов без возврата

---

### Граничные случаи

| Ситуация | Ожидаемое поведение |
|---|---|
| localStorage заполнен (90+ записей) | Старые удаляются, хранятся последние 90 |
| Поле заметки пустое | Запись сохраняется без заметки |
| Открыть вне Telegram | Работает, имя пользователя — заглушка |
| Нет интернета | Работает локально (localStorage), API недоступен |
| API недоступен при открытии | AI-чат работает в locked-режиме (aiAccessGranted = null) |

## Часть 3: Backend API

**Base URL:** `https://api.telo-pomnit.ru`

### Health check
```bash
curl https://api.telo-pomnit.ru/health
# → {"status":"ok","timestamp":"..."}
```

### Основные эндпоинты

| Метод | URL | Что делает |
|---|---|---|
| POST | `/auth/telegram` | Авторизация через initData |
| GET | `/me` | Профиль пользователя |
| GET | `/me/enrollment/access` | `{ hasAccess, canWrite }` |
| GET | `/diary` | Записи дневника |
| POST | `/diary` | Новая запись |
| GET | `/checkins/today` | Чекин за сегодня |
| POST | `/checkins` | Новый чекин |
| POST | `/ai-chat` | Сообщение AI-ассистенту |
| POST | `/create-payment` | Создать платёж ЮКасса → вернуть URL |
| POST | `/webhook/yukassa` | Webhook от ЮКассы после оплаты |
| POST | `/send-welcome` | Вручную отправить welcome-письмо |

### Проверить после деплоя на сервер

```bash
# На VPS:
pm2 list                          # telo-backend — online
pm2 logs telo-backend --lines 20  # нет ошибок
curl http://localhost:3000/health  # {"status":"ok"}
```

## Что делать при ошибке

| Симптом | Где смотреть |
|---|---|
| Белый экран в Mini App | F12 → Console → красные ошибки |
| Скрипт не загрузился | Порядок `<script>` в `tg-app/index.html` |
| Данные не сохраняются | F12 → Application → Local Storage → ключи `tp_*` |
| API возвращает 401 | JWT истёк или initData невалидна |
| Бот не отвечает | `pm2 logs telo-backend` на сервере |
| HTTPS не работает | `nginx -t && systemctl reload nginx` на сервере |
