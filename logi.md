### [2026-03-15] Сайт не открывается у других пользователей
**Симптом:** сайт открывался только у владельца (домашний WiFi), у всех остальных — не загружался
**Причина:** включена Vercel Deployment Protection — сайт был закрыт только для членов Vercel-команды
**Исправление:** Vercel Dashboard → Settings → Deployment Protection → отключить toggle → передеплой
**Коммит:** `004879e Redeploy: отключена Vercel Authentication`

---

### [2026-03-15] Яндекс.Метрика блокировала загрузку
**Симптом:** сайт медленно загружается
**Причина:** скрипт Метрики стоял в `<head>`, блокировал рендер страницы
**Исправление:** перенести `<script>` Метрики в самый конец `<body>`
**Коммит:** `1a32170 Fix: Яндекс.Метрика перенесена в конец body`

---

### [2026-03-15] Контент не виден до загрузки JS
**Симптом:** при отключённом JS или медленном соединении страница была пустой
**Причина:** `.reveal` элементы скрывались через `opacity:0` и ждали IntersectionObserver
**Исправление:** `.reveal{opacity:1;transform:none}` — контент виден по умолчанию, анимация добавляется классом `.animate` только через JS
**Коммит:** `5694aaf Fix: убрать все сломанные reveal-хаки`

---

### [2026-03-15] Loading screen блокировал скролл на мобильном
**Симптом:** на телефоне нельзя было листать страницу
**Причина:** `#loading-screen` с `position:fixed;inset:0` перехватывал touch-события
**Исправление:** добавить `pointer-events:none` в CSS для `#loading-screen`
**Коммит:** `d7e5483 Fix: loading screen не блокирует скролл`

---

### [2026-03-15] Hero секция обрезала контент на мобильном
**Симптом:** на телефоне часть содержимого hero (кнопки, таймер) обрезалась снизу
**Причина:** `.hero{align-items:center;overflow:hidden}` — при центрировании контент выходил за границы
**Исправление:** `@media(max-width:768px){ .hero{align-items:flex-start;padding-top:80px} }`
**Коммит:** `a53a322 Fix: hero не обрезает контент на мобильном`

---

### [2026-03-15] Страница не прокручивалась на iOS Safari
**Симптом:** на iPhone страница останавливалась на hero-секции, секции ниже недоступны
**Причина:** `body{overflow-x:hidden}` — в iOS Safari это блокирует вертикальный скролл (известный баг браузера)
**Исправление:** убрать `overflow-x:hidden` с `body`, обернуть весь контент в `<div style="overflow-x:hidden;position:relative">`
**Коммит:** `f8f5b43 Fix: overflow-x:hidden на wrapper div вместо body`
**Важно:** НЕ вешать `overflow-x:hidden` на `<html>` — тоже ломает скролл

### [2026-03-15] Силуэт девушки в hero почти не виден
**Симптом:** пользователи не видели декоративный силуэт тела в hero-секции
**Причина:** `.hero-silhouette{opacity:.1}` — 10% прозрачности на светлом фоне = невидимо
**Исправление:** `opacity:.1` → `opacity:.35`
**Коммит:** `37d3cfc Fix: силуэт девушки видимее`

### [2026-03-15] `<picture>` теги не растягивались на всю ширину
**Симптом:** изображения в галерее и секции ведущих отображались некорректно

### [2026-03-16] Сайт не открывался в России без VPN
**Симптом:** `https://telo-pomnit.ru/` не грузился в России, но работал через VPN и на `telo-pomnit.vercel.app`
**Причина:** A-запись домена указывала на старый Vercel IP `216.198.79.1` — один адрес, заблокированный у российских провайдеров. `vercel.app` использует несколько IP включая `64.29.17.3`, который доступен из России.
**Исправление:**
1. Сменить NS на рег.ру на Cloudflare: `amir.ns.cloudflare.com` + `jule.ns.cloudflare.com`
2. В Cloudflare DNS → A-запись `telo-pomnit.ru`: заменить `216.198.79.1` на `76.76.21.21` (актуальный Vercel IP)
3. Статус прокси для A и CNAME — **DNS only** (серое облако, не Proxied)
4. CNAME `www` → `cba612f53ea38a8d.vercel-dns-017.com` — DNS only
**Итоговые DNS записи в Cloudflare:**
- `A telo-pomnit.ru → 76.76.21.21` (DNS only)
- `CNAME www → cba612f53ea38a8d.vercel-dns-017.com` (DNS only)
- `A api → 45.11.93.236` (DNS only) — Beget VPS
**NS Cloudflare:** `amir.ns.cloudflare.com` / `jule.ns.cloudflare.com`
**Статус:** ✅ работает в России без VPN с 2026-03-16

### [2026-03-16] SMTP Яндекс не принимал пароль
**Симптом:** Nodemailer возвращал ошибку `Invalid user or password` при отправке письма
**Причина 1:** Яндекс не разрешает использовать основной пароль аккаунта для SMTP — нужен пароль приложения
**Причина 2:** IMAP/SMTP не был включён в настройках почты Яндекс
**Исправление:**
1. mail.yandex.ru → Настройки → Почтовые программы → включить «С сервера imap.yandex.ru по протоколу IMAP»
2. Яндекс ID → Безопасность → Пароли приложений → создать пароль «backend» для Почты (IMAP/POP3/SMTP)
3. Использовать полученный пароль приложения в `SMTP_PASS` в `.env`
**Файл:** `backend/.env` (SMTP_PASS), `backend/src/services/email.js`
**Важно:** пароль приложения показывается один раз — сразу сохранить в `.env`

### [2026-03-16] SSH-ключ для автодеплоя на Beget VPS
**Симптом:** Claude не мог деплоить бэкенд автоматически — SSH по паролю не работал
**Причина:** Beget по умолчанию использует нестандартный порт SSH или PasswordAuthentication=no; пароль не подходил
**Исправление:**
1. Сгенерировать SSH-ключ ed25519: `ssh-keygen -t ed25519 -f c:/tmp/beget_key`
2. Добавить публичный ключ в `/root/.ssh/authorized_keys` через Файловый менеджер Beget (VNC-терминал не поддерживает вставку)
3. Деплой теперь: `ssh -i c:/tmp/beget_key root@45.11.93.236 "cd /var/www/telo-pomnit && git pull origin main && pm2 restart all --update-env"`
**Файл:** `c:/tmp/beget_key` (приватный), `c:/tmp/beget_key.pub` (публичный)

## Известные особенности проекта

| Что | Как работает |
|---|---|
| Кэш HTML | `Cache-Control: no-store` в `vercel.json` — HTML не кэшируется на CDN |
| Reveal-анимации | `.reveal{opacity:1}` по умолчанию видимы; класс `.animate` добавляется JS для анимации появления |
| Loading screen | Скрыт через `style="display:none"` в HTML; JS показывает его на 1.5 сек |
| Изображения | Все используют `loading="lazy"` кроме hero-силуэта; форматы: webp + jpg/png fallback |
| Домен | `telo-pomnit.ru` → Cloudflare → Vercel → `landing_final.html` |
